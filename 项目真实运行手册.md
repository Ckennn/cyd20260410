# 项目真实运行手册

本文档以当前代码实现为准，覆盖 `main.py` 的真实运行方式、步骤依赖、输入输出文件与常见注意事项。

---

## 1. 当前主线

当前仓库的真实主线是：

**SQLite 数据 → 指标化 CSV → 个股信号 → 行业热度特征 → XGBoost 动态阈值 → Step 3 交易指令 → Step 4/5 回测**

主入口：

- `main.py`

核心说明：

- `main.py` 使用的是 **SQLite 版本**。
- **Step 0 每次运行 `main.py` 都会执行**。
- 当前最新版优化主线是 **XGBoost 动态阈值**。
- Step 4 个股回测已经接入：
  - 市场状态识别
  - 动态止损
  - Sig 专属卖出
  - 卖出原因归因

## 3. 环境准备

## 3.1 推荐环境

- Python 3.11
- Windows / PowerShell 更稳妥

原因：

- 仓库中部分脚本使用了 Windows 风格路径（反斜杠），在 Windows 环境下最稳。
- 在 WSL / Linux 下运行时，个别脚本可能需要手动调整路径常量。

## 3.2 虚拟环境

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

先安装仓库已有依赖：

```powershell
pip install -U pip setuptools wheel
pip install -r requirements.txt
```

再补齐当前主线代码实际用到、但 `requirements.txt` 未完整覆盖的依赖：

## 5. `main.py` 的真实运行规则

`main.py` 支持两种参数方式：

```powershell
python main.py 4 20240101 20241130
python main.py --start 20240101 --end 20241130
```

### 5.1 位置参数格式

```text
python main.py [步骤号] [开始日期] [结束日期]
```

例如：

```powershell
python main.py 3 20240101 20241130
```

### 5.2 可选参数

- `--skip-query`：跳过 Step 1
- `--skip-strategy`：跳过 Step 2
- `--skip-analysis`：跳过 Step 2.5 与 Step 3

### 5.3 很重要的真实行为

1. **Step 0 总会执行**
2. `python main.py 3 ...` **不会自动执行 Step 2**
3. `python main.py 4 ...` **不会自动执行 Step 3**
4. `python main.py 5 ...` **不会自动执行 Step 3**

所以：

- 单独跑 Step 3 前，必须先有 Step 2 的 trigger 结果
- 单独跑 Step 4 / 5 前，必须先有 Step 3 的交易指令文件

## 6. 推荐运行方式

## 6.1 全流程运行（最稳）

```powershell
python main.py 0 20240101 20241130
```

这会按顺序执行：

1. Step 0：生成行业参数
2. Step 1：导出日线数据与指标
3. Step 2：生成个股信号
4. Step 2.5：训练 XGBoost 动态阈值
5. Step 3：生成交易指令
6. Step 4：个股回测
7. Step 5：行业回测

## 6.2 复用已有 Step 1 数据，重跑后续步骤

```powershell
python main.py 0 20240101 20241130 --skip-query
```

适合：

- 数据缓存已存在
- 只想重跑策略 / XGBoost / 回测

## 8. XGBoost 主线的真实位置

当前主线接法：

1. Step 2 生成个股 trigger
2. `analyze_industry_heat.py` 聚合为行业热度历史
3. `train_xgboost_threshold_model.py` 滚动训练并输出 `prob_up`
4. `industryanalysis.py` 根据 `prob_up` 动态修改板块买入阈值
5. Step 3 输出交易指令
6. Step 4 / 5 间接受到 XGBoost 影响

当前阈值逻辑：

- `prob_up < 0.45` → 禁止买入
- `prob_up > 0.70` → 激进买入
- `prob_up < 0.30` → 倾向强制卖出/清仓

## 10. 常见问题

## 10.1 Step 3 跑不出来结果

先检查：

- 是否已经先跑过 Step 2
- `results_zh_*_trigger.csv` 是否存在于 `cache_files/debug/quantitativeResultsOfStocks/`

## 10.2 Step 2.5 报缺包

通常缺：

- `xgboost`
- `scikit-learn`
- `seaborn`

## 10.3 Step 4 报 `talib` 或回测相关错误

先检查：

- `ta-lib`
- `backtrader`

## 10.4 Step 0 报 Excel 读取错误

先检查：

- `C模型策略-1016.xlsx` 是否存在
- `openpyxl` 是否已安装

## 10.5 数据库路径不对

当前默认路径是：

- `data/stock_data.db`

如要改路径：

```powershell
$env:STOCK_DB_PATH="D:\your\path\stock_data.db"
python main.py 1 20240101 20241130
```
