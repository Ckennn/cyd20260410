# 项目优化行动指南 (Action Plan) - 动态阈值与板块效应

## 1. 核心目标
将交易策略从**“固定参数试错”**升级为**“动态博弈预测”**，利用**行业热度（个股Sig触发比例）**的周期性特征来指导交易。
- **买入逻辑**：确认**板块效应**形成（个股触发比例达到动态**起点阈值**且呈增长趋势），抓取确定性高的第二/三波行情。
- **卖出逻辑**：预测**顶点阈值**，结合**冷却/记忆机制**（Cooling Phase），在热度见顶回落后的下跌通道中坚决不买入，严控回撤 < 10%。

## 2. 关键调整点 (Key Adjustments)
根据最新会议纪要，对原计划进行如下微调：
- **特征精简**：**不再特意对龙头股 (lead_stock_count) 进行单独建模**。
    - *理由*：板块触发1-2个信号已隐含龙头启动；我们核心关注的是龙头是否带动了板块效应（即触发数量/比例的持续增加）。
- **阈值定义**：重点关注**“触发比例” (Trigger Ratio %)**，而非绝对数量。
    - *示例*：某板块个股总数 69，买入阈值设为 **16%**（约11只），且要求数量呈增长趋势。
- **动态预测**：重点预测**买入阈值 (Threshold_Buy)**，卖出逻辑结合动态顶点与趋势反转（如 T2, T3 连续下降）。

## 3. 实施步骤 (Execution Steps)

### Phase 1: 数据工程与特征构建 (Data Engineering) ✅ 已完成
**目标**：构建“行业热度历史数据表”，为分析提供基础。

1.  **输入数据**：
    - `trigger.csv`: 历史个股信号触发记录。
    - `stock_industry` 表: 获取每个行业的个股总数 (Total Count)。
    - `industry_quote` 表: 获取行业指数涨跌幅 (用于验证有效性)。
2.  **构建特征 (Feature Construction)**：
    - 按日、按行业统计：
        - `sig_count`: 当日触发信号个股数。
        - **`sig_ratio`**: `sig_count / total_stock_count` (核心指标)。
        - `sig_trend`: `sig_count` 较昨日的变化 (增长/持平/下降)。
3.  **输出产物**：
    - `cache_files/industry_heat_history.csv`

### Phase 2: 动态阈值预测模型 (Dynamic Threshold Model) ✅ 已完成
**目标**：实现“3个月滚动预测3天”的逻辑，生成动态参数。

#### XGBoost 模型深度解析 (Deep Dive into XGBoost)

##### 1. 模型类型
*   **本质**：二分类器 (Binary Classifier)。
*   **算法**：eXtreme Gradient Boosting (XGBoost)。这是一种基于决策树的集成学习算法，通过串行训练数百棵“弱分类器”（树），每一棵树都试图修正前一棵树的预测残差（错误）。
*   **优势**：在金融时序数据中，XGBoost 相比深度学习（神经网络）更抗噪，相比随机森林更能捕捉复杂的非线性模式（如：热度过高反而是顶部）。

##### 2. 输入特征 (Input Features X)
模型每天接收以下多维度信息作为“感知器”：
*   **热度维度**：
    *   `sig_ratio`: 当前行业有多少比例的个股触发了信号？（核心指标）
    *   `sig_trend`: 今天的热度比昨天是增加了还是减少了？（判断是在升温还是降温）
    *   `sig_ma5`: 过去5天的平均热度。（平滑短期噪音）
*   **价格维度**：
    *   `price_trend`: 行业指数当前偏离 MA20 均线多少？（判断是多头排列还是空头反抽）
    *   `roc_5`: 过去5天的累计涨幅。
*   **风险维度**：
    *   `volatility`: 最近5天的波动率。（判断市场是否处于极度恐慌或亢奋状态）

##### 3. 预测目标 (Target Y)
*   **定义**：**未来 3 天**该行业指数的累计涨幅是否 **> 2%**？
*   **标签**：
    *   `1` (True): 是，未来3天是上涨的（值得买）。
    *   `0` (False): 否，未来3天是下跌或震荡的（不值得买）。

##### 4. 运作流程 (Workflow)
1.  **滚动训练 (Rolling Training)**：
    *   每天收盘后，模型会“回忆”过去 90 天的所有历史数据。
    *   它学习这 90 天里，当 `sig_ratio`、`price_trend` 等特征处于什么组合时，未来 3 天真的涨了（Target=1）。
2.  **当日预测 (Prediction)**：
    *   输入今天的特征数据。
    *   模型输出一个 **概率值 (Prob_Up)**，例如 `0.85`。
    *   这意味着：模型认为基于今天的盘面特征，未来3天上涨的概率是 85%。

##### 5. 决策逻辑 (Decision Logic)
根据输出的概率值，动态调整交易系统的买入门槛：
*   **进攻模式 (`Prob_Up > 0.7`)**：
    *   模型非常确信会上涨。
    *   **动作**：将买入阈值降至极低（如 5%），只要有一点点信号就买入，不踏空。
*   **防守/冷却模式 (`Prob_Up < 0.4`)**：
    *   模型认为大概率要跌（可能是假突破或下跌中继）。
    *   **动作**：将买入阈值升至极高（如 999%），**强制空仓**，防止接飞刀。
*   **中性模式 (`0.4 <= Prob_Up <= 0.7`)**：
    *   **动作**：维持标准的买入阈值（如 15%）。

### Phase 3: 策略执行与风控 (Execution & Risk Control) ✅ 已完成
**目标**：在交易环节落实“记忆”与“冷却”机制。

1.  **状态机 (State Machine) 升级**：
    - 维护行业状态：`IDLE` -> `HEATING` (达到动态阈值) -> `HOLDING` -> `COOLING` (顶点确认回落)。
2.  **冷却机制 (Cooling Mechanism)**：
    - **规则**：若行业处于 `COOLING` 状态（刚从高点 `Threshold_Sell` 下来），即使当日 `sig_ratio` 再次短暂触及 `Threshold_Buy`，也**禁止开仓**。
    - **目的**：防止在下跌中继接飞刀，确保只在这一波行情的“左侧启动点”或“右侧确认点”介入，避开“右侧下跌段”。

### Phase 4: 回测环境升级 (Backtest Enhancement) ✅ 已完成
**目标**：解决固定金额回测导致的资金利用率低下问题。

1.  **动态仓位管理 (Dynamic Position Sizing)**：
    - **逻辑**：废弃固定金额（如 50000 元/次）的买入方式。
    - **实现**：引入百分比仓位控制，每只触发买入的 ETF 分配总资金的 **18%**（`target_value = total_value * 0.18`），最多允许持仓 5-6 只。这使得在复利效应下，资金能随净值增长而放大。
2.  **紧急逃顶落地**：
    - **逻辑**：在 Step 3 中，当 XGBoost 预测暴跌概率极高 (`prob_up < 0.30`) 时，动态修改 `sell_param` 为 `0.1/9999`，强制触发清仓指令，不再死等常规止损线。

## 4. 验证计划 (Validation)
1.  **相关性可视化**：绘制 `sig_ratio` vs `行业指数涨跌幅` 的散点图/热力图，确认“16%”等阈值的有效性。
2.  **回测对比**：
    - **基准**：使用当前固定参数的 Step 5 回测结果 (约 5% 收益)。
    - **实验**：使用动态阈值参数 + 动态仓位的 Step 5 回测结果。
    - **当前成果**：已成功将 2024.01-2024.11 的 ETF 累计收益率提升至 **26.73%** (胜率 43.66%)。
3.  **下一步优化 (Future Work)**：
    - **回撤控制**：当前最大回撤为 -21.15%，略高于大盘。计划引入大盘系统性风险过滤（如沪深300跌破MA60时，强制降低单只持仓上限至10%）。
    - **平滑阈值**：将阶梯式的阈值开关改为连续的线性函数映射。
